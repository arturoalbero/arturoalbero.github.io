# Práctica MD: Desarrollo de una Página Web usando GitHub Pages y Markdown

## Objetivo

El objetivo de esta práctica es desarrollar una página web colaborativa utilizando **GitHub Pages** y **Markdown**. Además, aplicar **Scrum** para la planificación y gestión del proyecto, así como **Git** para el control de versiones. Se evaluará tanto el resultado final de la web como el proceso de trabajo en equipo.

## Requisitos

Cada equipo de trabajo debe cumplir con los siguientes requisitos:

1. **Creación de una página web**:
   - La página puede estar compuesta por una sola página o por varias páginas interconectadas.
   - Los contenidos deben estar escritos en **Markdown**.
   - La página debe ser **funcional**, estar **disponible públicamente** y alojada en **GitHub Pages**.

2. **Planificación y gestión del proyecto**:
   - Debe realizarse una **planificación inicial** del proyecto que incluya los objetivos, las tareas y el tiempo estimado para cada una.
   - Se deben aplicar **metodologías ágiles** (como Scrum) para la gestión del trabajo.
   - Cada integrante del equipo debe estar a cargo de al menos una tarea o apartado de la web.

3. **Uso de Git y GitHub**:
   - El trabajo debe desarrollarse usando **Git** para el control de versiones.
   - Es obligatorio trabajar en **ramas separadas**: cada apartado de la web debe desarrollarse en una rama distinta.
   - Al finalizar, todas las ramas deben integrarse en la rama principal (**main** o **master**) para la entrega final.

4. **Memoria del proyecto**:
   - Se debe entregar una **memoria** que documente todo el proceso del proyecto. Esta debe incluir:
     - La **planificación** y **temporalización**.
     - Un resumen del uso de Git, incluyendo los **commits** más relevantes.
     - Un enlace al **repositorio** del proyecto en GitHub.
     - Una **valoración individual** del trabajo en grupo, reflejando la contribución de cada miembro.

5. **Temática de la página web**:
   El profesor asignará a cada equipo una temática concreta, con una serie de puntos clave a tratar. Las temáticas se encuentran en el anexo.

## Entregables

1. **Página web** alojada en **GitHub Pages**, accesible públicamente.
2. **Memoria del proyecto**, que incluya planificación, temporalización, resumen de Git y valoración individual.
3. **Enlace al repositorio** de GitHub donde se ha desarrollado el proyecto.
4. **Intervención en el foro**. Se habilitará un foro donde cada equipo publicará un enlace a su web. El resto de estudiantes puede comentar sobre puntos concretos de la web.

## Criterios de Evaluación

La evaluación de la práctica se basará en los siguientes aspectos:

1. **Planificación** (2 puntos)
   - Se evaluará la planificación inicial del proyecto y su adecuación a las metodologías ágiles.

2. **Temporalización** (2 puntos)
   - Se evaluará cómo el equipo ha distribuido y gestionado el tiempo a lo largo del proyecto.

3. **Uso de Git** (3 puntos)
   - Se valorará la calidad del uso de Git, incluyendo el número de commits, su claridad y cómo se han gestionado las ramas.

4. **Calidad de la página web** (2 puntos)
   - **Veracidad de la información** (1 punto): la página debe reflejar información correcta y bien documentada sobre la temática asignada.
   - **Calidad y originalidad del contenido** (1 punto): se evaluará tanto la presentación como la profundidad y originalidad del contenido.

5. **Valoración individual** (1 punto)
   - Cada estudiante debe hacer una valoración sobre su propio trabajo y el de sus compañeros, detallando cómo ha sido la colaboración dentro del equipo.

6. **Participación en el foro (opcional)** (1 punto)
    - Cada estudiante puede valorar en el foro las páginas web de los compañeros. El comentario tiene que ser relevante y constructivo para ser valorado de forma positiva, y se tiene que notar que se ha leído el trabajo.

## Estructura de la Memoria

La memoria debe incluir los siguientes apartados:

1. **Introducción**:
   - Breve descripción del proyecto y la temática asignada.

2. **Planificación del proyecto**:
   - Descripción de los objetivos y de las tareas principales asignadas.
   - Explicación de cómo se ha organizado el equipo y la distribución de tareas.

3. **Temporalización**:
   - Resumen del calendario del proyecto, indicando las fases y los hitos principales.

4. **Uso de Git y GitHub**:
   - Descripción del flujo de trabajo con Git, incluyendo el uso de ramas y la integración de los cambios.
   - Ejemplos de commits importantes y justificación de las decisiones tomadas durante el desarrollo.

5. **Enlace al repositorio**:
   - Incluye el enlace al repositorio público de GitHub donde está alojado el proyecto.

6. **Valoración individual**:
   - Cada miembro del equipo debe redactar una valoración personal sobre su trabajo y el trabajo en equipo.

---

## Recomendaciones

- **Asegúrate de poder usar Github Pages**: Lee detenidamente las instrucciones de GitHub Pages. Aprende cómo funcionan los enlaces entre diferentes páginas y haz pruebas antes de empezar a estructurar el contenido. No deberíais invertir más de uno o dos días en esto.
- **Documentación**: Asegurarse de que toda la información relevante esté bien documentada tanto en el código como en la memoria.
- **Markdown**: Utilizar todas las funcionalidades de Markdown para mejorar la presentación del contenido (listas, encabezados, tablas, enlaces, etc.).
- **Git**: Realizar commits frecuentes y con mensajes descriptivos para que el progreso sea fácil de seguir.
- **GitHub Pages**: Probar que la página esté publicada y accesible antes de la entrega final.
